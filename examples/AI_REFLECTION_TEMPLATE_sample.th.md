## 3. ฉันถาม AI ให้ช่วยอะไร

อธิบายสั้น ๆ ว่าถาม AI ให้ช่วยเรื่องอะไร

ห้ามวาง chat log ทั้งหมด

ตัวอย่าง:

- ฉันถาม AI ให้ช่วยอธิบายโจทย์
- ฉันถาม AI ให้ช่วย review แผนแรกของฉัน
- ฉันถาม AI ให้ช่วยหา bug ใน code
- ฉันถาม AI ให้ช่วยเสนอ test cases
- ฉันถาม AI ให้อธิบายว่าทำไมผล OJ จึงผิด

```text
ฉันถาม AI ให้ช่วย review แผนแรกของฉันสำหรับการอ่านจำนวนเต็ม n ตัวและคำนวณผลรวม ฉันยังถามด้วยว่าวิธีอ่าน input ของฉันจะทำงานหรือไม่ ถ้าตัวเลขถูกแบ่งอยู่หลายบรรทัด
```

---

## 4. AI ช่วยให้ฉันสังเกตอะไร

อธิบายแนวคิด ข้อผิดพลาด bug edge case หรือ concept ที่ AI ช่วยให้สังเกต

```text
AI ช่วยให้ฉันสังเกตว่า ถ้าใช้ input().split() เพียงครั้งเดียว code อาจผิดเมื่อจำนวนเต็ม n ตัวถูกเขียนแยกหลายบรรทัด AI แนะนำว่าควรอ่านตัวเลขต่อไปจนกว่าจะได้ครบ n ค่า
```

---

## 5. ฉันตรวจสอบหรือแก้อะไรด้วยตนเอง

อธิบายว่าก่อนส่งเข้า OJ คุณตรวจสอบ ทดสอบ แก้ไข หรือตัดสินใจอะไรด้วยตนเอง

ตัวอย่าง:

- ฉันทดสอบ case เพิ่มเติมใน VS Code
- ฉันตรวจ input/output format
- ฉันเปรียบเทียบคำแนะนำของ AI กับ problem statement ใน OJ
- ฉันแก้ code หลังจากเข้าใจ bug แล้ว
- ฉันไม่ใช้บางส่วนของคำแนะนำจาก AI เพราะไม่ตรงกับโจทย์

```text
ฉันกลับไปตรวจ problem statement ใน OJ อีกครั้ง และยืนยันว่า input อาจมีตัวเลขมากกว่าหนึ่งบรรทัด ฉันแก้ code ให้รับตัวเลขต่อไปจนกว่าจะได้จำนวนเต็มครบ n ค่า จากนั้นฉันทดสอบ case ปกติ case ที่มีตัวเลขติดลบ และ case ที่ตัวเลขถูกแบ่งอยู่หลายบรรทัด
```

## 6. ฉันได้เรียนรู้อะไร

เขียน 2-4 ประโยคเกี่ยวกับสิ่งที่ได้เรียนรู้จากโจทย์นี้และจากการใช้ AI

ให้เน้นการเรียนรู้ของตนเอง ไม่ใช่แค่คำตอบสุดท้าย

```text
ฉันได้เรียนรู้ว่า input format สำคัญ และตัวเลขอาจไม่ได้อยู่ในบรรทัดเดียวเสมอ ฉันยังได้เรียนรู้ว่าควรทดสอบ case ที่แตกต่างจาก sample input ด้วย AI มีประโยชน์ในการช่วยชี้ให้เห็น bug ที่อาจเกิดขึ้นในการอ่าน input แต่ฉันยังต้องตรวจสอบด้วย test cases ของตนเอง
```
